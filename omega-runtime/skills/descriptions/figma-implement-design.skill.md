# Implement Design

## Summary
Translates Figma nodes into production-ready code through a strict MCP workflow focused on 1:1 visual fidelity.

## Use When
- Use when the user wants a Figma frame or component implemented as code.
- Use when parity to Figma is a primary success condition.

## Do Not Use When
- Do not use without a Figma source or accessible MCP connection.
- Do not use for generic UI ideation or critique-only work.

## Role In Omega Workspace
- Documents the high-fidelity Figma-to-code path as an external capability reference.

## Inputs And Preconditions
- Requires a Figma URL or a supported desktop selection flow.
- Requires design context and screenshot capture before implementation.

## Outputs And Effects
- Produces implementation-ready guidance and code work grounded in the referenced node.

## Constraints And Non-Goals
- Does not bypass the ordered MCP workflow.
- Does not permit silent drift away from the source design.
