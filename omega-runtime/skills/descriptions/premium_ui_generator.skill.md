# Premium UI Generator

## Summary
Generates or refines UI with stronger hierarchy, calmer composition, and more disciplined premium presentation.

## Use When
- Use when the user wants implementation or refinement of an existing UI toward a stronger premium outcome.
- Use when critique findings need to be translated into better composition and hierarchy.

## Do Not Use When
- Do not use for critique-only or design-system-audit-only work.
- Do not use for source extraction from Figma.

## Role In Omega Workspace
- Documents the main implementation-oriented refinement skill for premium UI output.

## Inputs And Preconditions
- Requires existing UI, source structure, critique, or explicit design constraints.
- Works best when protected elements and active mode are already defined.

## Outputs And Effects
- Produces implementation-ready UI refinements grounded in structure rather than decorative excess.

## Constraints And Non-Goals
- Does not invent source facts or silently re-architect protected flows.
- Must respect the active mode and source authority.

## Notes
