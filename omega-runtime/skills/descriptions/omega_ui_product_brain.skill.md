# Omega UI Product Brain

## Summary
Orchestrates the order, mode, and guardrails for multi-skill UI and product-design work.

## Use When
- Use when one UI task needs coordinated critique, flow review, generation, and system enforcement.
- Use when protected elements and execution sequence must be defined before implementation.

## Do Not Use When
- Do not use for a narrow task that clearly needs only one specialized UI skill.
- Do not use outside UI, UX, product-design, or design-to-code work.

## Role In Omega Workspace
- Documents the coordination layer that chooses the right design-skill sequence and stop conditions.

## Inputs And Preconditions
- Requires a UI task with meaningful scope, constraints, or multiple possible skill paths.
- Works best when source truth or explicit product constraints already exist.

## Outputs And Effects
- Produces active mode selection, protected elements, skill sequencing, and drift-prevention rules.

## Constraints And Non-Goals
- Does not replace source extraction, critique, or generation on its own.
- Does not allow lower-priority taste changes to override stronger constraints silently.
