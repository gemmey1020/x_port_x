# God Plan Mode

## Summary
Builds execution-ready plans for ambiguous, risky, or multi-step coding work.

## Use When
- Use when planning, scoping, sequencing, or de-risking complex work.
- Use when a stricter plan contract or stronger verification path is needed.

## Do Not Use When
- Do not use for trivial direct execution with no planning need.
- Do not use as a substitute for implementation.

## Role In Omega Workspace
- Serves as a planning-reference skill for structuring implementation work around the workspace.

## Inputs And Preconditions
- Requires a task that benefits from explicit mission, facts, assumptions, strategy, and verification.

## Outputs And Effects
- Produces a structured execution plan with proof and risk handling.

## Constraints And Non-Goals
- Planning only; it does not perform the implementation itself.
- Should stay evidence-driven and concise.
