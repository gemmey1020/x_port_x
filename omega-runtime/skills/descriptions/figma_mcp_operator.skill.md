# Figma MCP Operator

## Summary
Governs Figma-based work by extracting implementation intent, labeling deviations, and verifying parity against the source.

## Use When
- Use when Figma-derived implementation needs explicit preserved-vs-changed framing.
- Use when design-to-code work must stay grounded and auditable.

## Do Not Use When
- Do not use for purely generative UI work with no Figma source.
- Do not use as a replacement for implementation or generic critique skills.

## Role In Omega Workspace
- Documents the operator layer that keeps Figma-based execution disciplined and mode-aware.

## Inputs And Preconditions
- Requires a Figma source and an active execution mode.
- Works best when downstream implementation or review will follow.

## Outputs And Effects
- Produces source-reading guidance, deviation labeling, and parity verification framing.

## Constraints And Non-Goals
- Does not replace the implementation skill itself.
- Does not silently override explicit source or product constraints.
