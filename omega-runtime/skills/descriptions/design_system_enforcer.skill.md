# Design System Enforcer

## Summary
Enforces repeatable UI system behavior across tokens, components, spacing, emphasis, states, and responsive rules.

## Use When
- Use when an existing interface needs consistency cleanup or stronger token discipline.
- Use when repeated UI patterns should behave like a system instead of one-off styling decisions.

## Do Not Use When
- Do not use for Figma source extraction.
- Do not use for product-flow redesign or pure visual critique.

## Role In Omega Workspace
- Documents the workspace rule-set for systemizing UI output after generation or review.

## Inputs And Preconditions
- Requires an existing interface, design direction, or concrete UI output to evaluate.
- Works best when button roles, card families, spacing patterns, or responsive behavior already exist.

## Outputs And Effects
- Produces system verdicts, drift points, and small consistency-preserving corrections.

## Constraints And Non-Goals
- Does not redesign the product flow.
- Does not invent a new design system unless explicitly requested.
