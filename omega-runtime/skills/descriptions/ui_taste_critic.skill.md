# UI Taste Critic

## Summary
Critiques UI quality using evidence-based judgment across hierarchy, spacing, typography, contrast, density, consistency, CTA clarity, and polish.

## Use When
- Use when the user wants a stronger visual review, more premium taste, or actionable polish feedback.
- Use when the interface needs prioritized critique before another implementation pass.

## Do Not Use When
- Do not use for Figma extraction, product-flow architecture, or direct implementation alone.
- Do not use when only system enforcement is needed.

## Role In Omega Workspace
- Documents the critique layer that identifies what weakens an interface and what should change first.

## Inputs And Preconditions
- Requires concrete UI output, screenshots, or implemented structure to judge.
- Works best when the visible hierarchy and component relationships can be inspected.

## Outputs And Effects
- Produces prioritized findings and visual-improvement guidance tied to visible evidence.

## Constraints And Non-Goals
- Does not flatter output or give vague praise.
- Does not replace generation or system-enforcement workflows.
