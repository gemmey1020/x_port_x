# Frontend Skill

## Summary
Shapes visually strong landing pages, websites, apps, and demos with deliberate hierarchy, art direction, restraint, and motion.

## Use When
- Use when the work needs a stronger visual thesis rather than more components.
- Use when the request targets a landing page, branded site, app surface, prototype, or demo UI.

## Do Not Use When
- Do not use for design-source extraction from Figma.
- Do not use for design-system cleanup or flow architecture as the primary task.

## Role In Omega Workspace
- Documents the external interface-generation skill that pushes UI work toward premium composition and away from generic layouts.

## Inputs And Preconditions
- Requires a UI task where composition, hierarchy, and presentation quality materially affect the result.
- Assumes the operator will still respect any existing design system or brand constraints in the workspace.

## Outputs And Effects
- Produces guidance for bold, restrained, image-led, motion-aware UI implementation.

## Constraints And Non-Goals
- Prioritizes layout, hierarchy, and art direction over feature sprawl.
- Does not replace source-based Figma workflows or formal design-system enforcement.
