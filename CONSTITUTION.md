# PORTFOLIO SPECTRUM: SYSTEM CONSTITUTION (V1.0)
## ARCHITECTURE & GOVERNANCE DOCUMENT

### 1. MISSION STATEMENT
To build a high-performance, scalable ecosystem that generates dynamic portfolios for Egyptian tradespeople, transforming traditional crafts into "Digital Brands." Every output must be executable, high-conversion, and localized for the Egyptian market.

### 2. CORE OPERATING PRINCIPLES
- **Arabic-First:** All user-facing content must be in clear, professional, yet accessible Egyptian Arabic.
- **Mobile-First (Strict):** Design for the 99% who use mobile devices. Assume low-end hardware, cracked screens, and non-standard aspect ratios.
- **Intelligent Responsive Design (IRD):** UI must remain readable and reliable on all screen sizes. Priority is given to content clarity over decorative elements.
- **No Simulation-Only Output:** Every strategy or persona generated must be ready for JSON schema mapping and technical implementation.

### 3. THE TIER SYSTEM (MODES)
Every trade must be derived into three distinct tiers:
1.  **LITE:** Minimalist, fast-loading, focusing on the core identity and a single WhatsApp CTA.
2.  **DEFAULT:** Standard professional portfolio including a services list, gallery, and work hours.
3.  **PREMIUM:** Full-featured brand experience with video anchors, advanced SEO, and high-end UI polish.

### 4. TECHNICAL STACK & CONSTRAINTS
- **Framework:** Next.js 16 (App Router).
- **Styling:** Tailwind CSS (Utility-first).
- **UI Components:** Shadcn/UI & Framer Motion for polishing.
- **SEO Logic:** Mandatory JSON-LD Schema implementation for Local SEO (Local Business / Professional Service).
- **Conversion:** Primary CTA is always "Direct to WhatsApp."

### 5. CONTENT & PERSONA RULES
- **Persona Generation:** Must be grounded in market reality. A plumber's persona is different from a carpenter's. 
- **The "Brag" Factor:** The UI must be impressive enough for the craftsman to show off to peers (Social Proof).
- **Dual-Path Synchronization:** - **Path A (Revival):** Social media engagement and page activity.
    - **Path B (Ecosystem):** The technical portfolio itself.

### 6. THE GROK VERIFICATION PROTOCOL
Before any strategy is finalized:
1. Generate the strategy/persona.
2. Formulate 3-4 aggressive, critical questions (The Grok Roast).
3. Verify via Grok to identify market flaws or "fluff" before production.

### 7. EXECUTION PROTOCOL
- **No state exists unless produced by execution.**
- **No UI without a real engine.**
- **Audit before trust. Freeze before scale.**
